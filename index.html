How it works:

What's kept:

  • • Host URLs and security schemes
  • • All endpoints with methods and paths
  • • Brief descriptions (first 100 chars)
  • • Parameter names, types, and requirements
  • • Request/response schema references
  • • Only referenced schemas with minimal fields
  • • Status codes and enum values

What's removed:

  • • Long descriptions (truncated to 100 chars)
  • • Examples and sample data
  • • Documentation and markdown
  • • Tags and external docs
  • • Verbose titles and summaries
  • • Unused schema definitions
  • • Pretty formatting and whitespace
